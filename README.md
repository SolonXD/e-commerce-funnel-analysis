# Retailrocket E-commerce Funnel Analysis
(Dataset можно скачать по ссылке: https://www.kaggle.com/datasets/retailrocket/ecommerce-dataset)

## О проекте
Этот проект посвящен анализу воронки покупок в e-commerce на основе event-level данных Retailrocket.

В рамках проекта исследуется путь пользователя от просмотра товара до покупки:
- `view`
- `addtocart`
- `transaction`

Цель анализа — определить, на каких этапах воронки происходит наибольшая потеря пользователей, а также выявить товары и категории с высоким интересом, но низкой конверсией.

---

## Цели проекта
- построить базовую и строгую пользовательскую воронку;
- рассчитать конверсии между этапами;
- оценить время до конверсии;
- сравнить поведение покупателей и непокупателей;
- проанализировать эффективность товаров и категорий;
- сформулировать продуктовые гипотезы для роста покупок.

## Данные
Источник данных: Retailrocket e-commerce dataset.

Используемые файлы:
- `events.csv`
- `category_tree.csv`
- `item_properties_part1.csv`
- `item_properties_part2.csv`

Основные поля таблицы `events.csv`:
- `timestamp` — время события;
- `visitorid` — идентификатор пользователя;
- `event` — тип события;
- `itemid` — идентификатор товара;
- `transactionid` — идентификатор транзакции.

---

## Основные этапы анализа

### 1. Data Cleaning and EDA
- загрузка и очистка событий;
- проверка структуры данных;
- анализ распределения событий;
- динамика активности пользователей во времени.

### 2. Funnel Analysis
- базовая пользовательская воронка;
- строгая последовательная воронка;
- конверсия `view → addtocart → transaction`;
- анализ времени до покупки;
- сравнение покупателей и непокупателей.

### 3. Item and Category Analysis
- товарные метрики по просмотрам, корзинам и покупкам;
- анализ конверсии по товарам;
- извлечение категорий из item properties;
- анализ категорий с высоким интересом и низкой покупкой.

---

## Ключевые метрики
- conversion rate `view → addtocart`
- conversion rate `addtocart → transaction`
- conversion rate `view → transaction`
- drop-off между этапами
- время до добавления в корзину
- время до покупки
- item-level conversion rate
- category-level conversion rate

---

## Основные результаты
- Наиболее массовое событие в данных — просмотр товара (`view`).
- Основные потери пользователей происходят на этапе перехода от просмотра к добавлению в корзину.
- Строгая последовательная воронка дает более консервативную и реалистичную оценку пользовательского пути, чем простая воронка по факту наличия событий.
- Распределение времени до конверсии имеет выраженную правостороннюю асимметрию: большая часть пользователей совершает целевые действия быстро, но присутствует длинный хвост более длительных решений.
- Среди товаров и категорий есть сегменты с высоким интересом, но низкой конверсией в покупку, что указывает на потенциальные точки роста продукта.
